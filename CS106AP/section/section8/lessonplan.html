Modules

We discussed the notions of modules in Python. A module (which is basically the formal name for any .py file), allows you encapsulate a particular group of variables and functions (and classes, although that's not relevant to 106A) such that they can be imported into another module. You can import prewritten modules -- such as math and urllib -- but can also define your own. For example, suppose you have the following code in mycoolmodule.py: 

          
def foo(n, m):
    # sick code here
    return n + 2

def bar():
    # more sick code here

You could leverage the module in another script, like so: 

        
import mycoolmodule  # must be same name as the .py file

def main():
    print(mycoolmodule.foo(42))
    mycoolmodule.bar()

if __name__ == "__main__":
    main()

List Comprehensions

List Comprehensions -- my personal favorite part of Python -- are a piece of syntactical sugar used to succinctly and expressively construct lists of elements. It's inspired by set builder notation set builder notation , a mathematical shorthand for expressing the elements of sets.

A list comprehension essentially produces a list by iterating over the elements of some collection, applying some (optional) operation to each element, possibly filtering the elements and adding them to a list. That's a lot to take in, but is all expressed quite succinctly using this general format:

[<operation(e)> for e in <original collection> <if predicate(e)>]

Where each of the parts in brackets (except the if predicate, which is optional based on whether or not you want to filter the elements) should be substituted for valid python code. For example, if you wanted to find the first 100 squares, you would do so as follows: 

        
>>> [i * i for i in range(100)]
[0, 1, 4, 9, 16, ..., 9801]       # lots of output elided

If you wanted to do the same thing for just the even numbers, you would adjust the comprehension by adding a predicate: 

        
>>> [i * i for i in range(100) if i % 2 == 0]
[0, 4, 16, ..., 9604]             # lots of output elided

You could also go wild and mix up your types: 

        
>>> d = {'Xyz': 50, 'abc': 42, 'xabc': 1000}
>>> [len(k) for k in d.keys() if k.lower()[0] == 'x']
[3, 4]

Those examples should hopefully provide you with enough of a working understanding of list comprehensions to teach section, but if you'd like to delve deeper into them (which I strongly recommend), I'd check out this fantastic tutorial a fantastic tutorial that I've linked here on the online version of this handout .

The absolute best thing for you to do here if you're unfamiliar with them is to actually work through the lecture examples as well as the section material, and to let Brahm know if you have any questions. This is also a really good refresher, if you need it. I've also linked a great refreseher on the online version of this handout, if you want to remind yourself of some of the nitty-gritty. THat said, I'll quickly provide a summary of the messaging you can use when talking about Jupyter notebooks:

Python, as a general purpose programming language, can be used in a surfeit of different contexts. One incredibly popular use is as an analytic tool, which allows even non-technical users to -- with only a small amount of easy-to-understand code -- engage with large amounts of data and to visualize it. A Jupyter notebook is a programming environment that lends itself particularly well to that kind of work.

You can think of a Jupyter notebook as a complete Python program, which is divided into sections by portions of text and images. The text can provide a summary of or context for the code you've written, and the images can be visualizations provided by your code. You are able to run each piece of code (called a 'cell') separately, so that you can iterate on individual parts of your code without having to rerun all of it every time (meaning, for example, you can do the time-intensive file reading once) and then quickly change the functions you call on that data.
